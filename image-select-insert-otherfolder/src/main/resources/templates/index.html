바탕화면에 이미지 저장 폴더 만들기

	사진을 저장하고 불러올 때 sql 컬럼 자료형으로 varchar2를 사용할 수 있고
	이미지 자체를 저장하는 Blob 을 사용할 수 있음
	db자체에 이미지를 저장하게 되면 이미지가 훼손할 가능성이 있거나 이미지를 글자로 변환할 때
	변환에 문제가 생길 가능성이 높음
	
	db 자체에 이미지를 저장하게 되면 비용이 많이 들기 때문에 많이 사용하지는 않는 방법
	
	이미지를 저장할 때 sql 자료형(varchar2)으로 이미지 경로를 설정해주고 
	설정한 이미지 경로를 가져오는 방식을 진행
	
	이미지가 저장되는 경로
	src/main/resources
				┗static  	= css/ js / font 와 같이 공개가능하고 유출가능한 코드
					┗		= 홈페이지에서 공개가 가능하고, 공통으로 사용할 수 잇는 이미지
	회사에서 지정한 이미지 저장경로 = 회사가 아닌, 고객이 올리거나 회사에서 비공개로 이미지를 가져오고
							저장하는 경로
	라이브러리나 프레임워크가 설정되지 않은 js 코드는 유출이 될 가능성이 높음
	라이브러리나 프레임워크로 설정된 js는 코드 형식이 라이브러리나 프레임워크의 형식으로 설정되어 있기 때문에
	외부에서 코드를 유추하기 어려움
	
	만약에 우리가 사진을 저장하길 원하는 폴더가 존재하지 않거나 의도치 않게 사라지거나 
	폴더 안에 이미지 이름과 형식이 모두 일치하여 기존의 이미지를 덮어씌우거나, 또는 이미지가 저장이 안될
	가능성이 있기 때문에 이미지를 저장할 때는
	1. 이미지를 불러오거나 저장할 경로가 존재하는 지 확인
		-> 경로가 존재하지 않아 불러오지 못할 때 : 이미지를 불러오는데 실패했습니다. 라는 이미지 를 띄워주기
			: 이미지를 불러오는데 실패했습니다. 라는 이미지를 띄워주기
		-> 이미지를 저장할 때 저장을 원하는 폴더가 존재하지 않음 
			: 이미지 경로 확인 원하는 경로가 없으면 경로를 생성 -> 이미지를 저장하기 위해
	2. 이름이 중복될 경우
		-> 이미지를 저장할 때 이름이 중복될 경우
			: 이미지를 원하느 ㄴ폴더에 저장할 때 기존에 사용하던 이미지 이름과 형식이 존재하여
			  이미지가 덮어 씌워지거나 저장이 안 될 경우 이름 중복을 피하기 위해
			  저장하는 시간을 뒤에 붙여서 저장해줌
